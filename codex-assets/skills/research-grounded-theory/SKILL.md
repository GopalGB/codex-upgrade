# research-grounded-theory

Grounded theory GENERATES theory from data rather than testing a prior hypothesis — so you deliberately delay the deep literature review to avoid forcing the data into existing frames. Code in escalating layers: **open coding** (line-by-line, naming phenomena), **axial coding** (relating categories to subcategories — conditions, actions, consequences), **selective coding** (integrating around a **core category** that ties the story together).

The engine is the **constant comparative method**: compare each new incident to prior incidents and codes, refining categories continuously. Collect and analyze data INTERLEAVED, not in phases — **theoretical sampling** means the emerging theory dictates who/what you sample next to develop weak categories. Write **memos** throughout (analytic notes capturing emerging relationships) — memos, not the codebook, are where the theory is built. Stop at **theoretical saturation**: new data adds no new properties to categories.

Know the variants: Glaser (emergence) vs Strauss-Corbin (structured coding paradigm) vs Charmaz (constructivist) — pick one and be consistent.

Pitfall: doing thematic analysis and calling it grounded theory — without theoretical sampling, constant comparison, and a core-category theory, it isn't GT. Second pitfall: forcing data into a pet framework instead of letting categories emerge.

**Tools:** open/axial/selective coding, constant comparative method, theoretical sampling, memo-writing, core category, NVivo/ATLAS.ti
