# career-engineer-portfolio-building

A portfolio's job is to prove you build *real, deployed, maintained* software and make sound tradeoffs. **2–3 deep projects beat 15 shallow ones.** Depth signals: tests + CI badge, a deployed live URL, an architecture diagram, a CHANGELOG, handled edge cases, and a README that explains *why* (tradeoffs, what you'd do differently) not just *how to run*.

Structure each README: one-line value prop → screenshot/GIF → architecture diagram → tech-stack rationale → setup → 'Engineering decisions' section (the differentiator). Pin 4–6 repos; write a GitHub profile README. For AI/ML roles, include an eval harness and a metrics table, not just a notebook. Deploy free: Vercel/Netlify (frontend), Fly.io/Railway (backend), HF Spaces (ML demos).

**What top candidates do differently:** they solve a real problem they personally have, contribute to OSS (merged PRs are third-party proof), and write a short blog post / postmortem per project — content multiplies a project's signal.

**Common mistake:** to-do apps and copied tutorial clones (zero signal — interviewers have seen 10,000), green-square farming with trivial commits, and dead demo links. A broken deploy URL is worse than no link. One polished, deployed, documented project outperforms a graveyard of half-finished repos.

**Tools:** READMEs with architecture diagrams, deployed demos, GitHub profile README, depth-over-breadth
