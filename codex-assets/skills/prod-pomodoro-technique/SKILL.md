# prod-pomodoro-technique

Pomodoro (Cirillo) fights procrastination by shrinking work to a non-threatening 25-minute commitment. Protocol: pick ONE task, set a timer for 25 min (one 'pomodoro'), work with zero interruptions, then take a 5-min break; after 4 pomodoros take a longer 15-30 min break. The pomodoro is **indivisible**-if you break it, it doesn't count, which trains single-tasking. Handle interruptions with the **inform-negotiate-schedule-call back** protocol for external ones, and for internal urges write the distracting thought on a capture sheet (an 'unplanned & urgent' line) and return to the task. Use pomodoros as an **estimation and measurement unit**: at day's start estimate how many a task needs; track actuals to calibrate future estimates and visualize total focused volume. The classic mistake is skipping breaks to 'stay in flow'-breaks are what make the cadence sustainable and prevent the late-day crash. Second mistake: a task needing 7+ pomodoros should be split; one needing under 1 should be batched with others. Best for fragmented schedules and aversive tasks; deep-work blocks suit longer uninterrupted flow.

**Tools:** Pomodoro, 25/5 cycles, internal/external interruption handling, pomodoro estimation
