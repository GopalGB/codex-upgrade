# prod-inbox-zero

Inbox zero (Mann) means an empty *inbox*, not an empty mailbox-the inbox is a processing queue, not a storage system or to-do list. Process in scheduled batches (2-3 times/day), never continuously. For each message apply **OHIO** (Only Handle It Once) and one of five Ds: **Delete/archive** (most email), **Delegate** (forward, then track on a Waiting-For list), **Respond** if it takes under 2 minutes (the 2-minute rule), **Defer** (snooze or convert to a task with a date-the action lives in your task system, not the inbox), **Do later/file** to reference. Pre-filter aggressively: rules that auto-archive newsletters/notifications into folders, unsubscribe ruthlessly, and route CCs out of the primary inbox so only true action items remain. The failure modes: (1) treating the inbox as a task list, so important actions rot under new mail-extract actions out; (2) checking continuously, which fragments focus all day-batch it; (3) over-organizing into 40 folders-search beats filing, so use 2-3 folders max. The metric is daily-to-zero at your last processing block, not perpetual emptiness.

**Tools:** inbox zero, OHIO, triage 5-D, email filters, batch processing, snooze/defer
