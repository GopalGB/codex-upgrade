# retail-assortment-planning

Assortment = breadth (categories/styles) x depth (SKUs per choice). Start by clustering stores (volume, climate, demographics) into 3-5 tiers so you don't ship swimwear to Alaska. Build a **choice count** per category per tier, then a **good-better-best** price-point architecture (e.g. opening $19 / core $39 / premium $79) ensuring no price gaps >35%.

The expert move: rank SKUs by trailing 52-week sell-through and GMROI, plot a Pareto curve, and cut the bottom tail where the bottom 20% of SKUs drive <5% of sales — but protect **traffic-driver** and **destination** SKUs even if low-margin. Use a **space-to-sales index** (% linear feet / % sales) to right-size facings; index >1.2 = over-spaced, candidate to cut.

Pitfalls: over-assorting (duplication confuses shoppers, raises markdowns), and pruning a 'low seller' that is actually a basket builder — always check attach rate before cutting. Validate the plan reconciles to the OTB dollar plan; an assortment that exceeds the buy budget is fiction. Localize the bottom tier, keep core consistent for supply-chain efficiency.

**Tools:** Excel (pivot tables, SUMIFS), space-to-sales index, 80/20 SKU ranking
