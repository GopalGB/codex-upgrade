# retail-markdown-optimization

Markdown = (original - new price)/original. The goal is **maximize total margin dollars recovered**, not minimize markdown %. A deep early cut that sells through often beats death-by-a-thousand-cuts that ends in salvage.

Use **sell-through triggers**, not the calendar: if at week 6 cumulative sell-through < target curve (e.g. <40% when plan said 55%), trigger markdown 1. Standard fashion cadence: first markdown ~25-30%, second 40-50%, final/clearance 60-75%, then liquidate. The expert move: model the **value of remaining inventory** — at each step compare expected margin of holding (sell more weeks at higher price) vs cutting now (faster cash, lower price). Aged inventory has carrying cost + opportunity cost of the OTB it ties up.

Pitfalls: marking down winners that would have sold at full price (margin leak), and waiting too long so the only buyers left are the most price-sensitive at near-zero margin. Permanent markdowns (PMD) for true clearance vs **temporary price reductions** (TPR/promo) for traffic — don't confuse them in reporting. Track **markdown cash recovery rate** and feed it back into next season's IMU so reductions are pre-funded.

**Tools:** markdown % formula, sell-through triggers, markdown cadence, GMROI recovery
