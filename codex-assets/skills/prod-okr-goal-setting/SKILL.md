# prod-okr-goal-setting

OKRs (Doerr/Grove) separate the inspiring direction (Objective: qualitative, time-bound, 'what') from the evidence of progress (3-5 Key Results: numeric, verifiable, 'how we'll know'). Write KRs as outcomes, not tasks-'ship feature X' is a task; 'increase activation rate from 40% to 55%' is a KR. **Score** each KR 0.0-1.0 at quarter-end; for *aspirational* (stretch) OKRs target ~0.7-hitting 1.0 means you sandbagged; for *committed* OKRs 1.0 is required. Cadence: set quarterly, review in weekly/biweekly check-ins with a confidence forecast (on-track/at-risk/off-track), grade and reflect at quarter close. Keep it to 3-5 objectives max-more dilutes focus. The two killing failure modes: (1) turning KRs into a task checklist so you measure activity not impact, and (2) tying stretch-OKR scores to compensation, which kills ambition because people set safe goals. OKRs are a focusing and alignment tool, not a performance-review weapon; cascade them by alignment and negotiation, not top-down dictation.

**Tools:** OKRs, key results, 0.0-1.0 scoring, aspirational vs committed, check-ins
