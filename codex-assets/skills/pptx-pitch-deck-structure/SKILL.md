# pptx-pitch-deck-structure

A deck is an argument, not a document. Use the canonical investor arc: Vision/Hook -> Problem -> Solution -> Why Now -> Product (demo/screens) -> Market (TAM/SAM/SOM) -> Business Model -> Traction (the proof slide) -> Competition -> Team -> Financials/Projections -> Ask & Use of Funds. Lead-with-traction if you have it. One idea per slide. Write the title as the takeaway (assertion-evidence: title = the claim 'Churn dropped 40% after onboarding redesign', body = the chart that proves it) — never topic labels like 'Metrics'. Kawasaki 10/20/30: ~10 slides, 20 minutes, 30pt-minimum font. Build the storyline in Sections first, fill last. Put detail (cohort tables, full financials, technical depth) in an Appendix after a 'Thank You / Appendix' divider so the core stays tight but you can jump there in Q&A (Section Zoom helps). Pitfall: the 'about us' opening, walls of bullets, and a market slide with a fake $50B TAM — investors discount all three. Each slide should answer 'so what?' in 3 seconds. Design supports the number; the number carries the slide.

**Tools:** Slide narrative frameworks (Guy Kawasaki 10/20/30, assertion-evidence); Sections; appendix pattern
