# research-qualitative-thematic-analysis

Follow Braun & Clarke's six phases: (1) familiarization — read all transcripts before coding; (2) generate initial codes line-by-line; (3) search for themes by clustering codes; (4) review themes against coded extracts AND the whole dataset; (5) define and name themes; (6) write up with verbatim evidence. Distinguish **codes** (a single feature, e.g. "fear of automation") from **themes** (a patterned meaning across codes, e.g. "identity threat").

Choose your stance up front: **inductive** (codes emerge from data) vs **deductive** (codes from a prior framework), and **semantic** (surface) vs **latent** (underlying assumptions). Build a codebook with code name, definition, inclusion/exclusion rules, and an example — this is what makes coding reproducible.

For positivist-leaning work, have a second coder code ~20% independently and report **Cohen's kappa** or **Krippendorff's alpha** (>= .70). For interpretivist work, reliability is replaced by an **audit trail** + reflexivity memos. Use NVivo/ATLAS.ti/Dedoose to manage, not to think.

Pitfall: "themes" that are just topic summaries or interview-question headings — a real theme makes an analytic claim. Second pitfall: cherry-picking quotes; report disconfirming cases.

**Tools:** Braun & Clarke 6 phases, NVivo/ATLAS.ti/Dedoose, codebook, Cohen's/Krippendorff's kappa, audit trail
