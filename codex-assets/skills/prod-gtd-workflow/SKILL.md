# prod-gtd-workflow

GTD externalizes every open loop so your brain stops re-surfacing them. Run five discrete steps, never blended: **Capture** everything into a small number of inboxes (one notes app, one physical tray, email). **Clarify** each item with the decision tree: Is it actionable? No -> trash, someday/maybe, or reference. Yes -> what is the *very next physical action*? If it takes under 2 minutes, do it now; else delegate (Waiting-For list) or defer. **Organize** deferred actions onto context lists (@calls, @computer, @errands) plus a Projects list for any outcome needing 2+ actions, and a calendar for date-specific items ONLY. **Reflect** via a Weekly Review: empty inboxes, review all lists, get current. **Engage** by picking actions filtered by context, time available, energy, and priority. The defining mistake: writing vague items like 'Mom' or 'taxes' instead of a concrete next action ('text Mom re: Thanksgiving dates'). Vague entries are why people abandon GTD. The calendar is sacred-only date/time-specific commitments go there, never a general to-do dump.

**Tools:** GTD, ubiquitous capture, next-action lists, contexts, tickler file
