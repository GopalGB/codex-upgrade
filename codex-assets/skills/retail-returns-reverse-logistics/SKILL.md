# retail-returns-reverse-logistics

Returns are a margin sink and a logistics flow in reverse. **Return rate = units returned / units sold**; apparel runs ~20-30%, e-comm higher than store, some categories 40%+. Each return carries reverse shipping, inspection labor, repackaging, and **value loss** (most returned goods can't be resold as new).

The expert move is the **disposition decision** per returned unit: **restock** (sellable, fastest recovery), **refurbish/repackage**, **discount/outlet**, **liquidate/jobber** (bulk salvage), or **destroy/recycle** (cost to dispose). Optimize for net recovery: a $40 item that costs $12 to refurb and resells at $25 nets $13 vs $8 liquidation — refurb wins; reverse it for low-value items where labor exceeds recovery, so destroy/donate.

Reduce returns at the source: accurate sizing/fit guidance, better PDP photos and reviews, and flagging **serial returners / return fraud** (wardrobing — wear-and-return, receipt fraud, empty-box). Track return reason codes to fix root causes (a SKU with 50% 'too small' returns needs a sizing fix, not more reverse logistics).

Pitfalls: auto-restocking without inspection (puts damaged goods back, generates re-returns), no reason-code discipline (can't fix what you don't measure), and ignoring return fraud (a small % of customers drive a large % of loss). Net the full reverse cost into category profitability — gross sales lie.

**Tools:** return rate, disposition decision (restock/refurb/liquidate/destroy), return fraud detection, RTV
