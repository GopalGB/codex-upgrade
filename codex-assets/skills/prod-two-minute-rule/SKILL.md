# prod-two-minute-rule

The 2-minute rule has two distinct uses-don't conflate them. **GTD version** (Allen): during clarify/processing, if a next action takes under ~2 minutes, **do it immediately** rather than capturing and organizing it, because tracking it would cost more time than doing it. This prevents the death-by-a-thousand-small-tasks buildup that clogs inboxes and lists. Apply it only while processing, not as an all-day interrupt-driven mode (that fragments focus). **Atomic Habits version** (Clear): to build a new habit, scale the entry action down to a 2-minute version-'read before bed' becomes 'read one page', 'do yoga' becomes 'put on yoga clothes'. The point is to master the *art of showing up*; once started, momentum usually carries you further, but you only need to commit to the 2 minutes. This defeats the activation-energy problem that powers procrastination. The failure mode for the GTD version is letting 'under 2 minutes' creep into 15-minute tasks that derail your processing pass; for the habit version, it's secretly expecting the 2 minutes to become 30-keep the bar genuinely tiny so it's non-negotiable on bad days.
