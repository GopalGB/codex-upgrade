# gsd-extract-learnings

Close the loop: a finished phase is a source of compounding knowledge. From the phase artifacts (SPEC/PLAN/diff/debug journal) extract: key DECISIONS (and why), LESSONS (what went wrong → the rule that prevents it — write these to `~/.codex/memory/LESSONS.md` / `.planning/LESSONS.md` per §F), reusable PATTERNS, and SURPRISES (assumptions that were wrong). This is how the kit gets smarter per project instead of repeating mistakes. Adapted from `gsd-extract-learnings`. Pitfall: skipping this means re-learning the same lessons next phase — the single highest-ROI 5 minutes in the loop. Cross-project patterns belong in the global LESSONS; repo-specific ones in `.planning/`. Next: `gsd-progress` to pick the next phase.

**Tools:** .planning/, LESSONS.md (§F)
