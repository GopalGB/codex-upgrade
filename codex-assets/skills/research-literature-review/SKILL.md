# research-literature-review

Build the search string first: core concept blocks joined by AND, synonyms within each block by OR, truncation (`econom*`) and phrase quotes (`"machine learning"`). Run it across at least 3 databases (Scopus, Web of Science, Semantic Scholar) plus one field-specific one — single-database reviews systematically miss ~30% of relevant work. Use backward snowballing (mine reference lists) and forward snowballing (cited-by, via Semantic Scholar / Connected Papers) to catch what keywords miss.

Screen in two passes: title/abstract against explicit inclusion criteria, then full text. Log every exclusion reason. The expert move is a **synthesis matrix** (rows = papers, columns = construct/method/sample/finding/limitation) so you synthesize ACROSS studies by theme, not summarize each paper serially. Code each paper for the gap it leaves.

Pitfall: the "annotated bibliography in disguise" — paragraph-per-paper with no argument. A real review organizes by themes/debates and ends every theme with the unresolved question your work addresses. Second pitfall: citation laundering — citing a claim from a secondary source without reading the primary. Always verify the original.

**Tools:** Boolean search operators, Google Scholar, Semantic Scholar, Connected Papers, Zotero, synthesis matrix
