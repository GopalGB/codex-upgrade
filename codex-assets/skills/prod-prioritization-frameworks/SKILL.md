# prod-prioritization-frameworks

Match the model to the decision's stakes. **RICE** = (Reach x Impact x Confidence) / Effort-best for product backlogs; Reach in count/period, Impact on a fixed scale (e.g., 3/2/1/0.5/0.25), Confidence as % to penalize guesses, Effort in person-months; higher score = do first. **ICE** = Impact x Confidence x Ease (1-10 each)-faster, lower-rigor, good for growth experiments. **MoSCoW** buckets scope into Must/Should/Could/Won't-best for fixed-deadline release scoping; the Won't list is the most valuable because it makes cuts explicit. **Weighted Scoring**: define criteria, assign weights summing to 100%, score each option, sum-best for multi-stakeholder decisions with competing dimensions. **Value-vs-Effort 2x2** for quick triage: do Quick Wins first, plan Big Bets, fill with Fill-ins, kill Time-sinks. The shared pitfall is **false precision**-treating fuzzy estimates as exact; always include a confidence factor and re-rank, don't compute once. Use the cheapest model that resolves the tie; reserve RICE/weighted scoring for high-stakes commitments.

**Tools:** RICE, ICE, MoSCoW, weighted scoring matrix, value/effort 2x2, WSJF
