# ba-storytelling-with-data

A chart is not a story — the narrative is. Use a three-act arc: setup (context, the metric and why it matters) → conflict (the surprising change or problem the data reveals) → resolution (what to do). Direct the eye with preattentive attributes: gray everything, then color/bold ONLY the one series or point that carries the message — the audience should look where you want in <1 second. Give every chart a declarative, takeaway title ('Onboarding drop-off doubled after the redesign'), not a descriptive label ('Activation by Week'); the title is the sentence you want remembered. Annotate directly on the chart — callouts, reference lines, a circled anomaly — rather than making people decode a legend. Reduce cognitive load: remove gridlines, borders, redundant axes, 3D, and chartjunk; one idea per slide. Expert moves: build to one Big Idea (a single declarative sentence with a point of view + stakes); sequence charts so each reveals the next question; use the 'so what?' test on every visual — if it has no action implication, cut it; tailor the framing to what the audience cares about (CFO=cash, PM=users). Pitfalls: showing all the data because you collected it; dual messages on one chart; decorative imagery that competes with the data; neutral titles that abdicate the interpretation; no clear recommendation at the end. Make them feel the conflict, then hand them the resolution.

**Tools:** storytelling with data, slide design, annotation
