# prod-time-blocking

Time-blocking assigns each task a specific calendar slot, converting an infinite to-do list into a finite plan that exposes overcommitment. Steps: (1) block fixed commitments first, (2) block 2-4 deep-work blocks of 60-120 min at your peak hours, (3) batch shallow/similar tasks (email, Slack, admin) into 1-2 dedicated windows-batching cuts context-switch cost since each switch carries an attention residue tax of up to ~20 min. Use **theme days** (e.g., Mon=planning, Tue/Wed=building, Thu=meetings) to reduce daily context churn. **Day-template**: a reusable ideal-day skeleton you copy weekly. Crucial discipline: estimate generously and leave 30-50% unblocked buffer-the universal failure mode is packing 8 hours of blocks into an 8-hour day with zero slack, so the first overrun cascades and you abandon the plan by 11am. When reality diverges, don't quit-do a 2-minute re-block of the remaining day. Cal Newport's rule: the goal isn't perfect adherence, it's intentionality; a wrong plan beats no plan because you can correct it.

**Tools:** time-blocking, task batching, theme days, day templates, calendar-as-system
